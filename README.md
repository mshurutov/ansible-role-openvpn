Task.
-----

create a role for deploying openvpn (may be other VPN services) that implements the following requirements:
* there are many VPN servers can be installed on one host;
* there are many VPN clients can be installed on one host;
* clients and servers can be installed and worked on one host at tne same time;

There are any restrictions:
* client names should be unique on one host;
* client should be configured by one config that contained keys and certs.

Requirements
------------

This role requires python v3 because python v2 is out of live.

Role Variables
--------------

Role has many variables. For details see defaults/main.yml

Dependencies
------------

This role use the tasks from follow files in role mshurutov.common:
* `prereqs.yml`, this file is used for check if necessary variables is defined;
* `ssl_keys.yml` and `ssl_keys_put.yml`, these files are used for generate and put SSL keys.

Using role
----------

### Install the roles.

Install roles on ansible contoller:

```
admin@ansible-controller ~ $ cd $ANSIBLE_ROLES_PATH
admin@ansible-controller roles $ ansible-galaxy role install mshurutov.common
admin@ansible-controller roles $ ansible-galaxy role install mshurutov.openvpn
```

### Full deploy

```
admin@ansible-controller prjname $ ansible-playbook ./vpn.yml
```

### Deploy openvpn to selected host or group

```
admin@ansible-controller prjname $ ansible-playbook ./vpn.yml -l vpnsrv
admin@ansible-controller prjname $ ansible-playbook ./vpn.yml -l vpnclients
```

### Create SSL keys

```
admin@ansible-controller prjname $ ansible-playbook ./vpn.yml -t openvpn_ssl_keys,create_ssl_keys
```

### Put SSL keys into place on OpenVPN servers

```
admin@ansible-controller prjname $ ansible-playbook ./vpn.yml -t openvpn_ssl_keys,put_ssl_keys
```

### Configure OpenVPN

```
admin@ansible-controller prjname $ ansible-playbook ./vpn.yml -t openvpn_config
```
